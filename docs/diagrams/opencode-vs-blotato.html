Who does what — opencode vs Blotato
You run the orchestration from opencode (or Claude Code) on efficient models. Blotato is the generation engine — it writes, renders, and publishes. The two talk over one REST client, scripts/blotato.py.

The point about modelsBecause opencode lets you pick any provider per agent, the whole orchestration side can run on cost-efficient non-Claude models (MiniMax, GLM, DeepSeek) — you only pay top-tier prices where judgment actually matters.
The point about BlotatoAll the expensive generation — copy, visuals, source extraction, hosting, scheduling, publishing — happens on Blotato's side through simple REST calls. You direct; Blotato produces.
